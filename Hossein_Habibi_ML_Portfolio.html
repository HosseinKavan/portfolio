★ Flagship Project · In Production

Two-Phase ML Network Optimization System

Phase 1: Hybrid HGNN + XGBoost (Learning-to-Rank) → Phase 2: Multi-Head HGNN

A city-scale, two-phase ML pipeline for automated 4G/5G RAN optimization. The system classifies network congestion, identifies the most suitable neighboring cells for offloading, and generates precise recommendations (azimuth, tilt, power adjustments) — all running in under 2 seconds for 1,088 cells. This is not a prototype; it is deployed in a live production network.

Phase 1 — Hybrid HGNN + XGBoost LTR

Unsupervised graph learning feeds into supervised ranking.

GraphSAGE on cell+sector graph → 64-dim node embeddings
Feature engineering: embeddings + UDCLI + KPIs + handover %
XGBoost Learning-to-Rank (LTR) · Pairwise/Listwise · NDCG objective
Custom UDCLI metric: load vs. capacity with MIMO/BLER correction
NBR Suitability Score = Weight% / UDCLI → ranked offload candidates
NDCG@10: 0.87 · MRR: 0.82 · Precision@5: 0.79
Phase 2 — Multi-Head HGNN (Behavior Cloning)

Transfer learning from Phase 1 initializes supervised prediction heads.

Shared GNN encoder (4× GraphSAGE layers + Phase 1 embeddings)
Head 1: Action Type — Azimuth / Tilt / CID (85% accuracy)
Head 2: Magnitude — ±1, ±2, ±3 adjustment (MAE: 1.2 units)
Head 3: Target Cell — uses Phase 1 rankings (78% accuracy)
Transfer learning → 40% faster convergence in Phase 2
Overall: 82% accuracy · Inference: <2 sec / 1,088 cells

Hybrid RAG Architecture for LTE Networks

Retrieval-Augmented Generation · Telecom Knowledge Engine

Designing an LLM-powered decision-support system tailored to telecom network data. The pipeline ingests structured KPI tables, unstructured engineer logs, and domain-specific documentation into a hybrid retrieval index — enabling intelligent question answering, root cause diagnosis, and optimization recommendations via natural language.

Architecture Overview

Hybrid indexing: Dense vector retrieval (FAISS) over embedded log narratives + sparse BM25 over structured KPIs. A re-ranker fuses both signals before passing context to an LLM fine-tuned on telecom domain vocabulary. Designed as an intelligent layer over the Phase 1/2 optimization system.

Network Trouble Ticket Classification & Anomaly Detection

Operational Intelligence for NOC Teams · City-Scale Network

An end-to-end ML pipeline applied to real telecom trouble ticket (TT) data across hundreds of sites. The analysis revealed that 98%+ of repetitive tickets originate from automated network surveillance systems generating false positives — drowning real operational issues. Key findings surfaced through classification, clustering, and duration analysis.

Key Findings from Real Data

127 critical sites generated 570 tickets (3% of sites = 10% of total volume). TCH/TCHA issues account for 55%+ of all tickets across 70% of unique sites. Sites like T2863 and T6667 each exceeded 400 repetitive tickets, indicating unresolved root causes masked by alert noise. Region R6 showed highest average ticket duration; R7 had highest volume. Handler bottleneck identified: only 3 groups processed the majority of tickets.

Reinforcement Learning · Energy

Energy-Saving RAN via Carrier Management with Reinforcement Learning

Q-Learning Agent · Huawei RAN · Dynamic Sleep Mode

A Q-learning RL agent that dynamically decides whether to keep or shut down sector carriers based on real-time user count. Trained over 1,000 episodes in a simulated RAN environment, the agent learns an optimal policy that maximizes energy savings while protecting network performance — turning off carriers only when fewer than 10 users are connected.

System Design

State space: 0–100 users per sector. Actions: keep carrier ON or switch OFF. Reward function: positive for energy-saving shutdowns under low load; negative for shutdowns during high demand. Epsilon-greedy exploration strategy. At 50 users: agent selects "keep on" with Q-value +5.2, demonstrating learned performance-aware policy.
